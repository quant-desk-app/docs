# Week of July 6, 2026

A quiet week focused on backend stability. No user-visible features changed.

## Updates

- **More reliable backend error reporting.** A sweep across deposits, positions, P&L validation, liquidation, conditional orders, order authorization, audit trails, and the WebSocket service tightens how errors and Redis calls are logged. The effect for you is fewer dropped error details and steadier behavior when something goes wrong on a request. See [API overview](/docs/developers/api-overview) and [Errors and rate limits](/docs/developers/errors-and-rate-limits).
- **Steadier WebSocket performance and integration tests.** Refreshed test coverage around the WebSocket service reduces the chance of regressions in live order, position, and balance streams reaching production. See [WebSocket overview](/docs/developers/websocket-overview).
