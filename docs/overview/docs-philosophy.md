# Docs philosophy

QuantDesk docs are written for high signal:

- Public docs explain usage, trust, and integration.
- Internal engineering history stays in repository-only docs.
- API schema truth stays in Swagger/OpenAPI; this site focuses on narrative and examples.

## What this avoids

- Dumping internal runbooks into public docs
- Mixing investor or scratch notes into user paths
- Copying legacy HTML dumps directly
